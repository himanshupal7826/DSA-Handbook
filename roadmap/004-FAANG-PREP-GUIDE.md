# 🏢 FAANG Interview Prep Guide

## What FAANG actually tests
- **Pattern recognition speed** — classify within 2 minutes.
- **Clean, bug-free coding** — templates you know cold.
- **Complexity fluency** — state and justify time/space before & after.
- **Communication** — narrate the brute→optimal journey.
- **Edge cases & testing** — empty, single, duplicates, overflow.

## The 80/20 of patterns
These show up most often:
1. HashMap Lookup (02), Frequency Counter (01)
2. Two Pointers (06–12)
3. Sliding Window (13–19)
4. Binary Search incl. on answer (20–28)
5. BFS/DFS + Topo + Union-Find (61–64)
6. Heaps / Top-K (43–47)
7. DP: Knapsack, LIS, LCS, grids, state machine (74–83)
8. Intervals (29–33) and Monotonic Stack (34–39)
9. Trees: DFS, BFS, LCA, tree DP (53–60)
10. Backtracking (69–73)

## Interview-day framework (the "R.E.A.C.T.O." loop)
1. **R**epeat / clarify the problem and constraints.
2. **E**xamples — walk a small input by hand.
3. **A**pproach — brute force first, then optimize out loud.
4. **C**ode — clean, using a known template.
5. **T**est — dry-run + edge cases.
6. **O**ptimize — discuss complexity & follow-ups.

## Level expectations
| Level | Bar |
|-------|-----|
| SWE / L3 | Solve 1–2 Mediums cleanly with optimal complexity |
| Senior / L5 | Mediums fast + 1 Hard, discuss trade-offs & scaling |
| Staff / L6 | Hard + system-level reasoning, ambiguity, multiple approaches |
