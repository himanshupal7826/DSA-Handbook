# 🗺️ DSA Patterns Handbook — Master Roadmap

This handbook teaches **100 patterns**, not 1000 problems. Master a pattern and you can solve every problem that hides inside it. Patterns are grouped from foundational to expert.

## How to use this handbook
1. **Pick a path** below based on where you are.
2. For each pattern: read *Recognition Signals* → *Optimal Approach* → memorize the *Template* → solve the *LeetCode Practice Set*.
3. Mark patterns **Complete** on their page; track progress on the home dashboard.
4. Re-run the **Pattern Selector** on any new problem until recognition becomes instant.

## The 15 sections of every pattern
Overview · Recognition Signals · Brute Force · Optimal Approach · Templates (Go/Python/Java/C++) · Complexity · Common Mistakes · Follow-Ups · 3 Solved Examples · LeetCode Set · Variations · Production Applications · Revision Notes.

## Golden rules
1. **Recognition before implementation.** Spend 2 minutes classifying before coding.
2. **One template per pattern.** Internalize it cold; adapt at the edges.
3. **Always state complexity** before and after optimizing.
4. **Dry-run small inputs** by hand — it catches 90% of bugs.
5. **Re-derive, don't memorize.** Understand *why* the optimal works.
